Troubleshooting Session One - HTML

Introduction

Essential Steps

  1. Edit HTML document in ATOM (or another text editor).
  2. Commit changes in GitHub Desktop (ensure you commit the changes to the correct branch)
  3. Push changes up to your website by selecting "Push origin".

Helpful pointers

Make sure your HTML files are saved with the .html extension.

Spaces in your file names can cause issues with GitHub Pages, and your files may not publish.

You can view your HTML files on your local computer by selecting File > Open With > (your web browser).
